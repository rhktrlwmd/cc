/ Информация о проекте

CAFECORE.

Цель проекта

Создать единую, удобную и интуитивно понятную систему, которая автоматизирует процессы продаж и обеспечивает точный учет остатков сырья и ингредиентов в режиме реального времени, минимизируя потери от просрочки и нехватки запасов.

Задачи разработки

  • /Разработка модуля POS для быстрого оформления заказов.
  • /Внедрение списания ингредиентов по факту продажи (рецептуры).
  • /Система автоматических уведомлений о дефиците.
  • /Реализация аналитической отчетности (топ-продукты, средний чек).
/ Документация

Техническое задание (ТЗ)

+
1. Общие сведения Система предназначена для комплексной автоматизации операционной деятельности кафе. Основная цель — эффективный учет продаж и запасов для минимизации потерь.
2.1. Модуль Продаж (POS) Быстрое создание заказа, выбор продуктов из каталога, автоматический расчет суммы, поддержка скидок, наличный и безналичный расчет. Фиксация истории продаж с фильтрацией.
2.2. Модуль Склада (Inventory) Учет ингредиентов и сырья, интерфейс создания рецептур, автоматическое списание при продаже. Система инвентаризации и оповещений о критическом остатке.
3. Требования к надежности Сохранение данных при сбоях, разграничение прав (Кассир/Менеджер), отклик системы < 1с.

/ Mathematics

Математический аппарат

+

1. Расширенная Сеть Петри (Логика списания)

Описывает параллельное списание ингредиентов:

PN = [P, T, F, W, M0] P = {P1: Заказ_Оплачен, P2: Рецептура_Найдена, P3: Остатки_Обновлены} T = {T1: Поиск_Состава, T2: Выполнение_Списания} M0(P1) = 1 (активация при каждой продаже)

2. Модель СМО (Очереди)

Анализ пропускной способности системы:

  • • $\lambda$ — Интенсивность входного потока клиентов.
  • • $\mu$ — Интенсивность обслуживания (скорость POS + Бариста).
  • • Цель: $P_{отк} \to 0$ (минимизация отказов в обслуживании).
